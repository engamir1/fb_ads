## القسم الأول: الملخص التنفيذي

### 1.1 رؤية المنتج
Facebook Ads Multi-Platform Manager هو نظام إدارة إعلانات متكامل يهدف إلى تمكين المعلنين من التحكم الكامل في حملاتهم الإعلانية عبر واجهة واحدة شاملة، تربط بين قوة Facebook Marketing API وسهولة الاستخدام اليومي. يهدف النظام إلى إزالة الحاجة للتبديل بين واجهات متعددة وتوفير رؤية 360 درجة لكل أنشطة الإعلان.

الهدف النهائي: تطبيق يعمل على جميع المنصات (Web, Desktop, Mobile, CLI, MCP) ويتيح التحكم الكامل في كل مستوى من مستويات هرم الإعلانات من الحساب الإعلاني حتى أصغر Creative Asset.

### 1.2 المشكلة التي يحلها
- تشتت بيانات الحملات عبر حسابات متعددة بدون مركزية.
- صعوبة مقارنة الأداء بين الحملات والمجموعات الإعلانية في لحظة واحدة.
- غياب أدوات Bulk Actions فعّالة لتعديل حملات كثيرة دفعة واحدة.
- ضعف الرؤية التحليلية العميقة في واجهة Ads Manager الافتراضية.
- صعوبة إدارة الإعلانات من الأجهزة المحمولة.

## القسم الثاني: بيانات Facebook API — التغطية الكاملة

يعمل Facebook Marketing API وفق هيكل هرمي من 5 مستويات. فيما يلي البيانات المتاحة في كل مستوى:

### 2.1 المستوى الأول: Ad Account (الحساب الإعلاني)
- **المعرف (ID)**: المعرف الفريد للحساب.
- **الحالة (Account Status)**: (نشط، معطل، بانتظار، إلخ).
- **المالية**: الرصيد الحالي (Balance)، إجمالي الإنفاق (Amount Spent)، الحد الأقصى للإنفاق (Spend Cap).
- **الإعدادات**: العملة، المنطقة الزمنية، مصدر التمويل.

### 2.2 المستوى الثاني: Campaign (الحملة الإعلانية)
- **الهدف (Objective)**: (Conversions, Reach, Lead Gen, App Installs, etc.).
- **الحالة**: (Active, Paused, Deleted, Archived).
- **الميزانية**: ميزانية يومية (Daily) أو كلية (Lifetime).
- **الجدولة**: وقت البدء والانتهاء.

### 2.3 المستوى الثالث: Ad Set (المجموعة الإعلانية)
- **الاستهداف (Targeting)**:
  - العمر، الجنس، اللغات.
  - المواقع الجغرافية (دعم الدول، المدن، الرموز البريدية).
  - الاهتمامات والسلوكيات والديموغرافيا.
  - الجماهير المخصصة (Custom Audiences) والمشابهة (Lookalike).
- **المواضع (Placements)**: (Facebook Feed, Instagram Reels, Audience Network, Messenger, etc.).
- **التحسين (Optimization)**: هدف التحسين (Optimization Goal) وحدث الفوترة (Billing Event).

### 2.4 المستوى الرابع: Ad (الإعلان الفردي)
- **المحتوى (Creative)**: الصورة، الفيديو، النص، الروابط.
- **الحالة**: (Active, Paused, Disapproved, Pending Review).
- **التتبع**: (Pixel tracking specs).

### 2.5 المستوى الخامس: Insights (مؤشرات الأداء)
- **المقاييس الأساسية**: (Impressions, Reach, Frequency, Clicks, Spend).
- **النسب**: (CTR, CPC, CPM).
- **التحويلات**: (Actions, Action Values, ROAS).
- **التقسيمات (Breakdowns)**: حسب العمر، الجنس، الجهاز، الموضع، الوقت.

## القسم الثالث: الخطة المستقبلية والميزات الكاملة

### 3.1 هيكل التطبيق (Architecture)
- **Core Library**: مكتبة TypeScript مشتركة للتعامل مع API.
- **State Management**: Zustand + React Query.
- **Web App**: Next.js 14 + TailwindCSS + shadcn/ui.
- **Desktop App**: Tauri v2 (استهلاك أقل للرام).
- **Mobile App**: React Native + Expo.
- **CLI Tool**: Node.js للأتمتة والعمليات الضخمة.
- **MCP Server**: للتحكم عبر الذكاء الاصطناعي (Claude).

### 3.2 خارطة الطريق (Roadmap)
- **المرحلة الأولى**: واجهة ويب للقراءة فقط مع تحليل البيانات (Insights).
- **المرحلة الثانية**: تفعيل عمليات الكتابة (إنشاء وتعديل الحملات) والـ Bulk Actions.
- **المرحلة الثالثة**: بناء محرك الأتمتة والقواعد الذكية (Rules Engine).
- **المرحلة الرابعة**: التوسع لمنصات أخرى (Google Ads, TikTok Ads) ودعم الوكالات.

## القسم الرابع: المواصفات التقنية

### 4.1 الـ API Endpoints الأساسية
- `/me/adaccounts`: جلب الحسابات.
- `/act_{id}/campaigns`: إدارة الحملات.
- `/act_{id}/adsets`: إدارة المجموعات.
- `/act_{id}/ads`: إدارة الإعلانات.
- `/act_{id}/insights`: جلب التقارير.

### 4.2 الصلاحيات المطلوبة (Permissions)
- `ads_read` & `ads_management`
- `business_management`
- `leads_retrieval`

## القسم الخامس: تصميم تجربة المستخدم

### 5.1 مبادئ التصميم
- **البيانات أولاً**: عرض الأرقام المهمة فوراً.
- **العمليات الجماعية افتراضياً**: كل جدول يدعم الـ Bulk Selection.
- **دعم الاختصارات**: للتحكم السريع من الكيبورد.
- **الوضع المظلم**: لراحة العين أثناء العمل لساعات طويلة.

## القسم السادس: الحالة الراهنة (MCP Server)

تم بناء سيرفر MCP حالياً يدعم العمليات القرائية التالية عبر Claude:
- جلب الحسابات وتفاصيلها.
- جلب الحملات والأد ستس والإعلانات.
- جلب الـ Insights لكل المستويات مع دعم التقسيمات (Breakdowns).
- جلب سجل التغييرات (Activities).

## القسم السابع: Business Manager والأصول (Assets)

### 7.1 إدارة Business Manager (BM)
- جلب حسابات الأعمال التي يديرها المستخدم.
- التحقق من حالة توثيق الحساب.

### 7.2 الأصول المرتبطة
- **الصفحات (Pages)**: جلب الصفحات واستخدامها كأصل للإعلان.
- **حسابات انستغرام**: ربطها لعرض الإعلانات هناك.
- **واتساب للأعمال**: لإعلانات الرسائل.

## القسم الثامن: التتبع المتقدم والكتالوجات

### 8.1 Pixels & Conversions API (CAPI)
- دعم قراءة بيانات الـ Pixel وأحداثه.
- التخطيط لدعم CAPI لإرسال الأحداث مباشرة من السيرفر لتفادي حظر الكوكيز.

### 8.2 الكتالوجات (Product Catalogs)
- جلب الكتالوجات واستخدامها في الإعلانات الديناميكية (DPA).
- قراءة الـ Product Sets لتقسيم المنتجات.

## القسم التاسع: خطة الهيمنة الشاملة للتطبيق

لتحقيق السيطرة الكاملة وأدق تحكم، سيتم بناء الآتي:

### 9.1 محرك القواعد الذكي (Rules Engine)
قواعد مخصصة تعمل على سيرفراتنا لتجاوز حدود فيسبوك:
- إيقاف الحملة فوراً إذا حققت CPA أعلى من رقم معين.
- زيادة الميزانية في الساعات الأكثر تحقيقاً للمبيعات تلقائياً.

### 9.2 دعم المنصات الكامل
- **CLI**: لكتابة سكريبتات تنشئ مئات الحملات في ثوانٍ.
- **Desktop**: للعمل في الخلفية وإرسال إشعارات نظام فورية عند حدوث أي خلل في الحساب.
- **Extension**: لتحليل إعلانات المنافسين مباشرة من المتصفح.
